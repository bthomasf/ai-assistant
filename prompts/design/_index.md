# 需求分析与方案设计 Prompt

## 需求分析

### 需求拆解
```
请分析以下产品需求文档，提取：
1. 核心功能点和优先级排序
2. 技术难点和风险点
3. 需要与产品确认的模糊点
4. 建议的任务拆解（含工时预估）

[粘贴 PRD 内容]
```

### 需求边界识别
```
基于以下需求描述，请帮我识别：
1. 显式需求（明确提到的功能）
2. 隐式需求（未明说但必须支持的功能）
3. 非功能性需求（性能、安全、兼容性等）
4. 潜在的需求冲突

[粘贴需求描述]
```

## 方案设计

### 技术方案生成
```
基于以下需求，请设计技术方案，包含：
1. 整体架构设计（含架构图描述）
2. 核心模块划分和职责
3. 数据库表结构设计（含索引建议）
4. 关键接口定义（RESTful）
5. 技术选型理由
6. 潜在风险和应对策略

技术栈：Java 17 + Spring Boot 3 + MyBatis-Plus + MySQL 8 + Redis
需求：[粘贴需求]
```

### 数据库设计
```
根据以下业务模型设计 MySQL 表结构：
1. 给出完整的 DDL
2. 设计合理的索引（含联合索引）
3. 说明字段设计理由
4. 考虑后续扩展性

业务模型：[描述业务实体和关系]
预估数据量：[填写]
```
