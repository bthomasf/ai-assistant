# 阶段七：问题排查与故障处理

## 阶段描述

线上问题定位与修复。AI 在日志分析和根因定位方面能提供很大帮助。

## 典型任务

| 任务 | 说明 |
|------|------|
| 日志分析 | 解读异常堆栈和错误日志 |
| 性能排查 | 分析慢查询、线程 dump、GC 日志 |
| 故障复盘 | 撰写故障复盘报告 |
| 应急脚本 | 生成数据修复/回滚脚本 |

## 推荐 AI 工具

| 工具 | 用法 | 推荐度 |
|------|------|--------|
| Bug-trace Skill | 根据 bug 追溯变更根因 | ★★★★★ |
| Mlog MCP | 搜索线上日志 | ★★★★★ |
| ChatGPT | 分析异常堆栈 | ★★★★☆ |
| Sourcegraph MCP | 搜索相关代码 | ★★★★☆ |

## 实用 Prompt 示例

### 异常分析
```
以下是线上报错的异常堆栈，请分析：
1. 根本原因是什么
2. 可能的触发条件
3. 建议的修复方案
4. 需要排查的相关代码

[粘贴异常堆栈]
```

### 慢查询优化
```
以下 SQL 执行时间超过 5 秒，请分析执行计划并优化：
表信息：[表名、数据量、现有索引]
SQL：[粘贴 SQL]
执行计划：[粘贴 EXPLAIN 结果]
```
