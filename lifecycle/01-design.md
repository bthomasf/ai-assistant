# 阶段一：需求分析与方案设计

## 阶段描述

从产品需求文档到技术方案落地的过程。这是开发工作的起点，好的需求理解和方案设计能大幅减少后续返工。

## 典型任务

| 任务 | 说明 |
|------|------|
| 需求理解与拆解 | 阅读 PRD，识别功能边界和模糊点 |
| 技术方案设计 | 选择架构模式，设计核心流程 |
| 数据库表结构设计 | 根据业务模型设计表结构和索引 |
| 接口契约定义 | 定义 RESTful API 或 RPC 接口 |
| 工时评估 | 拆解任务并估算复杂度和工时 |

## 推荐 AI 工具

| 工具 | 用法 | 推荐度 |
|------|------|--------|
| Claude | 长文档分析、方案生成 | ★★★★★ |
| ChatGPT | 需求梳理、方案对比 | ★★★★★ |
| Cursor Chat | 基于已有代码设计新方案 | ★★★★☆ |
| Overmind MCP | 查看需求详情和任务拆解 | ★★★★☆ |

## 实用 Prompt 示例

### 需求分析
```
请分析以下产品需求文档，提取：
1. 核心功能点
2. 技术难点和风险
3. 需要确认的模糊点
4. 建议的任务拆解

[粘贴 PRD 内容]
```

### 技术方案设计
```
基于以下需求，请设计技术方案，包含：
1. 整体架构设计
2. 核心模块划分和职责
3. 数据库表结构（含索引建议）
4. 关键接口定义
5. 技术选型理由

技术栈：Java 17 + Spring Boot 3 + MyBatis-Plus + MySQL 8 + Redis
需求内容：[粘贴需求]
```
