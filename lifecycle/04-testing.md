# 阶段四：测试与质量保障

## 阶段描述

单元测试、集成测试和质量度量。AI 能显著降低编写测试用例的成本。

## 典型任务

| 任务 | 说明 |
|------|------|
| 单元测试编写 | 为业务代码编写测试用例 |
| 测试数据构造 | 生成边界值和异常场景数据 |
| 接口测试 | 编写接口自动化测试 |
| 覆盖率分析 | 分析未覆盖路径并补充 |

## 推荐 AI 工具

| 工具 | 用法 | 推荐度 |
|------|------|--------|
| Cursor Agent | 批量生成测试用例 | ★★★★★ |
| GitHub Copilot | 根据方法签名补全测试 | ★★★★☆ |
| ChatGPT | 设计测试策略 | ★★★★☆ |

## 实用 Prompt 示例

### 单元测试生成
```
请为以下 Java 方法生成单元测试，使用 JUnit 5 + Mockito：
1. 覆盖正常流程
2. 覆盖边界条件
3. 覆盖异常场景
4. Mock 外部依赖

[粘贴方法代码]
```
