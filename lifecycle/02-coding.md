# 阶段二：编码开发

## 阶段描述

日常编码实现阶段，占工作时间比例最大。AI 工具在这个阶段能带来最直接的效率提升。

## 典型任务

| 任务 | 说明 |
|------|------|
| 业务逻辑编写 | 实现核心业务代码 |
| 模板代码生成 | CRUD、DTO、Mapper 等样板代码 |
| SQL 编写与优化 | 复杂查询编写和执行计划分析 |
| 正则/脚本编写 | 复杂正则表达式、Shell 脚本 |
| 重构 | 优化代码结构，消除技术债 |

## 推荐 AI 工具

| 工具 | 用法 | 推荐度 |
|------|------|--------|
| Cursor Agent | 多步骤自主编码、批量重构 | ★★★★★ |
| GitHub Copilot | 实时代码补全 | ★★★★☆ |
| DeepSeek | 代码生成（高性价比） | ★★★★☆ |
| GitLab MCP | 搜索参考实现 | ★★★★☆ |
| Sourcegraph MCP | 跨仓库代码搜索 | ★★★★☆ |

## 实用 Prompt 示例

### CRUD 生成
```
请基于以下表结构生成完整的 CRUD 代码，包含：
1. Entity 实体类
2. Mapper 接口（MyBatis-Plus）
3. Service 接口和实现
4. Controller（RESTful 风格）
5. DTO（请求和响应分离）

表结构：[粘贴 DDL]
```

### SQL 优化
```
以下 SQL 在大数据量下执行较慢，请分析并优化：
1. 指出性能瓶颈
2. 建议添加的索引
3. 提供优化后的 SQL

表数据量：约 500 万行
当前索引：[列出已有索引]
SQL：[粘贴 SQL]
```
