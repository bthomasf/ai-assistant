# 阶段三：代码审查

## 阶段描述

提交 MR/PR 前后的代码质量把关。AI 可以作为第一轮审查者，提前发现潜在问题。

## 典型任务

| 任务 | 说明 |
|------|------|
| 自审代码 | 提交前的自我检查 |
| 审查他人代码 | 帮助理解变更意图 |
| 代码规范检查 | 检查团队编码规范 |
| Commit 信息规范 | 生成规范化的提交信息 |

## 推荐 AI 工具

| 工具 | 用法 | 推荐度 |
|------|------|--------|
| Code Review Skill | 自动审查 git diff | ★★★★★ |
| Cursor Chat | 选中代码请求审查 | ★★★★☆ |
| Commit Skill | 生成规范 commit message | ★★★★☆ |
| GitLab MCP | 查看 MR 变更详情 | ★★★★☆ |

## 实用 Prompt 示例

### 代码审查
```
请审查以下代码变更，重点关注：
1. 逻辑错误和潜在 bug
2. 性能问题（N+1 查询、内存泄漏等）
3. 安全隐患（SQL 注入、XSS 等）
4. 代码规范违反
5. 可读性和可维护性

[粘贴 diff]
```
